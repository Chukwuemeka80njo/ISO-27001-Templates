# 08_Improvement

This folder contains all documentation related to corrective actions, continual improvement, and nonconformity handling as required by ISO/IEC 27001:2022.

Improvement activities ensure that weaknesses, failures, audit findings, and incidents are addressed systematically and that the ISMS continually becomes more effective.

---

## This section includes:

- Identification of nonconformities  
- Corrective action processes  
- Root cause analysis  
- Tracking of corrective and preventive actions (CAPA)  
- Opportunities for improvement  
- Continual improvement documentation  
- Evaluation of the effectiveness of corrective actions  

---

## Included Templates

- Nonconformity Report  
- Corrective Action Request (CAR) Form  
- Root Cause Analysis Template (5-Why / Fishbone / RCA)  
- CAPA Tracker  
- Continual Improvement Log  
- Improvement Opportunity Register  

---

## ISO/IEC 27001:2022 References

- **Clause 10.1** – Nonconformity and corrective action  
- **Clause 10.2** – Continual improvement
